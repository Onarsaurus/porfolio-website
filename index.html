Dream Tasker

A web application using JSPs and Java. Allows users to schedule events, use lists, mark notes, and even manage their finances, all in one website. The ultimate Swiss army knife of task management.

Overview

Introducing the Dream Tasker, the be all and end all of task managment! Designed to be the ultimate Swiss army knife of managment, this web application provides users with an easy to use and visually appealing calendar, lists, notes, budgeting, and more! For anyone who is tired of using mutliple tabs or pieces of paper for keeping track of their day to day responsibilities, take a rest and dream on in this one solves all solution.

My process

Before any code was written I used Lucid Charts to create class diagrams and database structure diagrams (UML). Once I had enough to work with and a solid vision I began creating the classes I needed and the front end, creating the main pages I would need for basic functionality like logging in and logging out. With that, I also needed a database and used my diagram to create it and work with. Generally, I tried to avoid css and thematic design, as I wanted to focus on the core functionality. My development for this project has unfortunately been a little all over the place, jumping from feature to feature, but I began with figuring out how to go about the calendar. Eventually I decided to use Calendar.js. From then on I wanted to work with lists. It was difficult to figure out the database side of working with lists but eventually I was able to get to an ok spot with the feature. Then I worked on a simple profile page where you could change your username and password. Throughout this whole process, I have asked the people around me what ideas they have and if they thought I could do something better, which has been a big help during the process.

I've learned a lot from this project, but to keep it short here are the main things I've learned (or learned more about) worth noting: database storage, servlets, JavaScript and back end communication, CSS, UX.

Project Cannonball

A web application GUI for interacting with a physical “cannon” that uses a queue of commands to execute its movements and actions. Uses JSPs for the GUI, Java, and Python.

Overview

Project Cannon ball describes a project in which the user uses a web GUI to interact with a physical cannon (arduino, servos, motors, etc). The user has access to commands and a queue which will be used to make the cannon move or fire. It should be noted this cannon is probably not what you are picturing. The user can can also create custom commands and these commands will be saved for later (Custom commands are just a preset queue of commands, you cannot technically create a new action for the cannon). Project Cannonball was the final project for my PC Capstone class.

Unfortunately

I don't have access to the physical cannon used by this project, however the code can still be analyized and inspected to see the intention.
